# Daily Recovery Coach 0.21.0

- Version: 0.21.0
- Maturity: local pre-1.0 release

## Summary

This release centers text and numeric values across every App table, resolves
Polar Dynamic API numeric sport identifiers through the official cached sport
catalog, and makes each sleep sync explicitly check the local current date.
Nightly Recharge requests include Polar's `features=samples`; when the summary
respiration interval is absent or zero, the App calculates the mean of valid
`breathingRateValues` samples for that night.

Nutrition history now includes deterministic, locally calculated estimated
calories and a category-coverage nutrition balance score for every meal event.

The exercise editor omits its redundant section heading, renders duration as
HH:MM:SS, and applies confirmed local labels for sport IDs 121, 83, and 15.
The recovery editor is fixed to today's record and omits measurement time; saving
both morning values starts the canonical manual sync so Polar sleep and today's
derived data refresh without waiting for the 06:00 fallback.

The current exercise and current sleep editing sections have now been removed in
full. Both pages begin with read-only history. Exercise durations, sleep clock
times and durations, duration baselines, and nutrition meal times render as
HH:MM:SS. Exercise heart rate, calories, fat percentage, and lowest sleep heart
rate use explicit parenthesized unit labels.

Nightly Recharge sample retrieval now follows Polar's one-day feature constraint:
the App fetches range summaries, requests `features=samples` separately for each
sleep date, merges samples by date, and then imports the observed breathing-rate
average. Baseline respiration is never substituted for a missing nightly value.

Read-only history tables now render as escaped local HTML so visible headers and
cells are both genuinely centered. Tables retain horizontal scrolling and sticky
headers; numeric values are limited to four decimal places with trailing zeros
removed.

The Recovery summary now uses the same centered HTML renderer instead of the
Canvas data editor whose leading edit icons visually offset column titles. A
separate expandable form preserves manual RMSSD/resting-heart-rate editing and
the existing post-save synchronization.

Nutrition item tables now combine quantity and unit into one `剂量（克）`
column. New and edited meal-item dosages use grams consistently, and the former
standalone unit column is no longer shown. The Canvas editor is replaced by
centered native inputs so column headings, item names, quantities, meal date, and
meal time are all visibly centered.

## Data Safety

No schema migration is required. Polar sport catalogs remain local raw-cache
files, and nutrition analysis is calculated from normalized meal items without
modifying source entries. Existing device and manual records remain unchanged.

## Verification

The release covers centered column configuration, sport catalog resolution,
current-date sleep detail checks, breathing-sample fallback, nutrition calculations, localization, and the
full regression suite. It is a personal tracking tool and is not a medical device.
